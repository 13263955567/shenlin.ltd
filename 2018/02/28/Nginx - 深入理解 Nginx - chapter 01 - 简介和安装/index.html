Nginx - 深入理解 Nginx - chapter 01 - 简介和安装

发表于 2018-02-28 | 分类于 Nginx

Nginx 由 Igor Sysoev 使用 C 语言开发。

安装要求

软件包

Nginx 的模块可根据需求来定制，完成 web 服务器最基本功能所需的模块：

查看 linux 内核版本

  • 内核需 2.6 版本以上，因 2.6 以上才支持 epoll

GCC

  • 使用源码编译安装 Nginx 时需要 gcc
  • 使用 C++ 开发 Nginx 模块时需要 gcc-c++

PCRE

  • perl compatible regular expressions
  • 配置文件 nginx.conf 中有时会使用正则表达式，需要调用 pcre 来解析
  • pcre-devel 是编译 nginx 必须的，也是使用 pcre 二次开发必须的

zlib

  • 用于对 HTTP 内容做 gzip 格式的压缩

OpenSSL

  • 使用 SSL 协议或 SHA1、MD5 等散列函数

配置 linux 内核

linux 内核参数考虑的是最通用的场景，并不适合作为高并发 web 服务器，最通用的支持高并发请求的 TCP 网络参数设置：

config 选项和参数

路径相关选项

参数 含义 默认值
–prefix=PATH 安装目录，会影响其他参数的相对目录 /usr/local/nginx
–sbin-path=PATH 可执行文件的相对路径 ≶prefix>/sbin/nginx
–conf-path=PATH 配置文件的相对路径 ≶prefix>/conf/nginx.conf
–error-log-path=PATH 核心错误日志文件的相对路径 ≶prefix>/logs/error.log
–http-log-path=PATH 访问日志文件的相对路径，每一个 HTTP 请求在访问结束后都会写入访问日志 ≶prefix>/logs/access.log
–pid-path=PATH pid 文件的存放路径，pid 文件里保存着 master 进程 ID ≶prefix>/logs/nginx.pid
–lock-path=PATH lock 文件的存放路径 ≶prefix>/logs/nginx.lock
–builddir=PATH 执行 configure 时和编译时临时文件的存放目录，包括产生的 Makefile、目标文件、可执行文件等 <nginx source path>/objs
–with-perl-modules-path=PATH perl modules 的路径，只有使用了第三方的 perl modules，才需要设置此路径
–with-perl=PATH perl 执行文件路径，配置 nginx 需执行 perl 脚本才需要设置此路径
–http-client-body-temp-path=PATH HTTP 请求的 request body 如果需要暂时保存则保存到此磁盘文件 <prefix>/client_body_temp
–http-proxy-temp-path=PATH Nginx 作为反向代理服务器时，上游服务器的 reponse body如果需要暂时保存则保存到此磁盘文件 <prefix>/proxy_temp
–http-fastcgi-temp-path=PATH fastcgi 使用的临时文件的存放目录 <prefix>/fastcgi_temp
–http-uwsgi-temp-path=PATH uwsgi 使用的临时文件的存放目录 <prefix>/uwsgi_temp
–http-scgi-temp-path=PATH scgi 使用的临时文件的存放目录 <prefix>/scgi_temp

编译相关选项

参数 含义 默认值
–prefix=PATH 安装目录，会影响其他参数的相对目录 /usr/local/nginx
